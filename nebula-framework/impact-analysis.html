3 Architecture & Pipeline

The framework implements a state-machine orchestrator driving agents through gated pipelines — matching the article's description of "Plan → Push back → Design → Push back → Build → Review → Fix → Verify → Ship".

Key Architecture Decisions

DecisionRationaleArticle Reference
State machine orchestratorDeterministic execution with clear state transitions"The workflow lived in the developer's HEAD - fragile and unrepeatable"
Agent contracts with boundariesPrevents role drift — PM cannot write code, Architect cannot change requirements"Contracts are: versioned, reviewable, reusable, improvable"
Gate validation between stepsCatches coordination failures before they propagate"Where Things Break: NOT syntax errors — COORDINATION failures"
Event-driven observabilityEvery state transition is recorded and streamable"The only honest way to validate is to make it build real work"
LLM provider fallback chainStrategy pattern: Azure → OpenAI → Mock. Zero-config demo mode"Tool-agnostic" — framework works regardless of LLM backend

6 Gap Analysis & Mitigation

EXERCISE 2 — Gaps Identified & Addressed
LOW
C# .NET Backend (nebula-insurance-crm uses C#)

Framework is language-agnostic — contracts are TypeScript types that map 1:1 to any typed language. Controller/Service/Repository pattern implemented here is identical to ASP.NET Core. Node.js patterns transfer directly.

LOW
Insurance Domain Knowledge

Mock LLM generates insurance-relevant artifacts (policies, claims, submissions). Healthcare workflow automation at Northwell Health provides direct pattern transfer — policy lifecycle mirrors patient care workflows.

LOW
Multi-Agent Framework Construction

Fully mitigated — this deliverable IS a working multi-agent framework with typed contracts, gated execution, LLM integration, and real-time observability. Not a concept; working software.

MED
Knowledge Graph Tooling

KG in nebula-insurance-crm is canonical-nodes + ontology in YAML/JSON — structured data alignment, not a graph database. Can extend gate engine to validate ontology sync (add a custom validator).

LOW
Authentik Identity Provider

Authentik is a Docker-based IdP with OAuth2/OIDC. Standard auth middleware pattern. Not implemented in framework (out of scope for exercise) but trivially addable via Express middleware.
