1. Project Overview

We built a multi-agent AI workflow system that demonstrates how AI agents can collaborate following a structured rulebook to automatically generate full-stack applications. The system showcases the concept behind MSIG's "Nebula" framework — where AI agents take on specific roles (Product Manager, Architect, Developer, QA) and follow a gated pipeline to produce software.

Key Concept: Instead of manually guiding AI through the same workflow every session, we define the workflow ONCE as code. Agents follow rules, produce artifacts, and pass quality gates before the next step begins.

2. What We Built

2.1 Agent Workflow Engine (Backend)

A Node.js/Express system with 4 AI agents working in sequence:

Product Manager → 🚪 Gate → Architect → 🚪 Gate → Developer → 🚪 Gate → QA → ✅ Feature Complete
Agent RoleResponsibilityProducesCannot Do
Product ManagerDefine WHAT to buildrequirements.json, stories.jsonWrite code, pick technology
ArchitectDesign HOW to build itapi-design.json, data-model.jsonChange requirements, write code
DeveloperWrite the actual coderoutes, controllers, modelsChange requirements/architecture
QATest everything workstest-results.jsonChange code or requirements

2.2 The Rulebook

A configuration file that defines:

  • Roles — What each agent can and cannot do
  • Workflow — The ordered steps (1→2→3→4)
  • Gates — Quality checkpoints between each step (validation functions)

2.3 Generated Application

When the agent pipeline runs, it automatically generates:

  • tasks-routes.js — A fully functional Express REST API (GET, POST, DELETE)
  • server.js — Express server with CORS, API endpoints for rulebook/build-summary
  • build-summary.json — Metadata showing what was built and why

2.4 React Frontend (Viewer UI)

A React + Vite frontend application with 3 views:

TabPurpose
Rulebook & WorkflowVisual pipeline diagram, agent roles, gates
Tasks Manager (Live)Live CRUD interface against the generated Express API
Build SummaryShows user stories, endpoints, generated files, test results

6. Key Takeaways

  • Why agents? — Eliminates repetitive AI sessions; workflow is codified once and reused
  • Why gates? — Prevents coordination failures (e.g., code that compiles but doesn't match the spec)
  • Why roles/boundaries? — Each agent can ONLY do its job, preventing scope creep and ensuring separation of concerns
  • Real-world application: Based on MSIG's Nebula framework for enterprise software delivery
